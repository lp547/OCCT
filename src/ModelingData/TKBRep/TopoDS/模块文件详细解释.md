# TopoDS 模块文件详解

本目录包含 Open CASCADE 拓扑数据结构的核心定义。TopoDS 定义了描述形状（Shape）及其相互关系的基础类。

## 核心基础类 (Core Base Classes)

### TopoDS.hxx
**描述**：这是 TopoDS 包的头文件。它包含全局辅助函数，用于安全地将通用的 `TopoDS_Shape` 向下转型（Cast）为具体的子类（如 `TopoDS_Face`, `TopoDS_Edge` 等）。如果类型不匹配，它会抛出异常。它是使用此包的入口点。

### TopoDS_Shape.hxx / .cxx
**描述**：这是整个拓扑系统的核心类。`TopoDS_Shape` 是一个轻量级的“句柄”（Handle），它引用一个底层的几何容器（`TopoDS_TShape`）。它主要包含三个信息：
1. 对底层拓扑结构的引用。
2. **Location（位置）**：一个局部坐标系变换。
3. **Orientation（方向）**：该形状相对于其父级的方向（如向前、向后）。

### TopoDS_TShape.hxx / .cxx
**描述**：代表“Topological Shape”（拓扑形状）。它是所有拓扑实体的基类（抽象类）。与 `TopoDS_Shape` 不同，`TShape` 存储了实际的子形状列表（如一个 Wire 存储了一组 Edges）。它是通过引用计数被共享的。

### TopoDS_Builder.hxx / .cxx / .lxx
**描述**：用于构建拓扑结构的工具类。它提供了自底向上（Bottom-Up）构建形状的方法，例如将 Edge 添加到 Wire 中，或将 Face 添加到 Shell 中。它负责检查拓扑结构的有效性（例如，不能将 Solid 添加到 Edge 中）。

### TopoDS_Iterator.hxx / .cxx
**描述**：用于遍历形状子元素的迭代器。当你需要访问一个 Wire 中的所有 Edge，或者一个 Solid 中的所有 Shell 时，就需要使用这个类。它会自动处理子形状的坐标变换（Location）和方向（Orientation）的叠加。

## 具体拓扑实体类 (Concrete Topological Entities)

这些类分为两组：以 `TopoDS_` 开头的句柄类（用户主要使用的），和以 `TopoDS_T` 开头的底层实现类（内部使用的）。

### TopoDS_Compound.hxx / TopoDS_TCompound.hxx / .cxx
**描述**：`Compound` 是最通用的容器，它可以包含任意类型的形状。通常用于将一组不相关的形状组合在一起处理。

### TopoDS_CompSolid.hxx / TopoDS_TCompSolid.hxx / .cxx
**描述**：`CompSolid`（复合实体）是由共享面连接的一组 Solid。它用于描述复杂的组合体结构。

### TopoDS_Solid.hxx / TopoDS_TSolid.hxx / .cxx
**描述**：`Solid`（实体）表示一个闭合的三维体积，由一个或多个 Shell（壳）包围。通常包含一个外壳和可能的内壳（空腔）。

### TopoDS_Shell.hxx / TopoDS_TShell.hxx / .cxx
**描述**：`Shell`（壳）是由一组连接的 Face（面）组成的表面。它可以是闭合的（形成 Solid 的边界）或开放的。

### TopoDS_Face.hxx / TopoDS_TFace.hxx / .cxx
**描述**：`Face`（面）是三维空间中的二维拓扑实体。它由一个底层的几何曲面（Surface）和限制该曲面的 Wire（线框）组成。

### TopoDS_Wire.hxx / TopoDS_TWire.hxx / .cxx
**描述**：`Wire`（线框）是由一组连接的 Edge（边）组成的轮廓。它通常用于定义 Face 的边界。

### TopoDS_Edge.hxx / TopoDS_TEdge.hxx / .cxx
**描述**：`Edge`（边）是三维空间中的一维拓扑实体。它对应于几何上的曲线（Curve），由两个 Vertex（顶点）界定。

### TopoDS_Vertex.hxx / TopoDS_TVertex.hxx / .cxx
**描述**：`Vertex`（顶点）是零维拓扑实体。它对应于几何上的点（Point）。

## 列表与集合 (Lists and Collections)

### TopoDS_ListOfShape.hxx
**描述**：定义了存储 `TopoDS_Shape` 对象的链表。这是管理子形状集合的标准方式。

### TopoDS_ListIteratorOfListOfShape.hxx
**描述**：用于遍历 `TopoDS_ListOfShape` 的迭代器。

### TopoDS_HShape.hxx / .cxx / .lxx
**描述**：`HShape` 是 `TopoDS_Shape` 的“Handle”封装版本。OCCT 的某些旧算法或特定接口需要通过 Handle（智能指针）来传递对象，这个类就是为了这种兼容性而存在的。

## 异常与警告 (Exceptions and Alerts)

### TopoDS_FrozenShape.hxx
**描述**：异常类。当试图修改一个已经被“冻结”（Frozen）或共享的形状时抛出。

### TopoDS_LockedShape.hxx
**描述**：异常类。当试图修改一个被锁定的形状时抛出。

### TopoDS_UnCompatibleShapes.hxx
**描述**：异常类。当试图将不兼容的子形状添加的父形状中（例如将 Solid 添加到 Edge）时抛出。

### TopoDS_AlertAttribute.hxx / .cxx
**描述**：用于在数据处理或传输过程中附加警告或错误信息的属性类。

### TopoDS_AlertWithShape.hxx / .cxx
**描述**：一种特定类型的警告，它将警告信息与特定的形状关联起来，方便用户定位问题发生的几何位置。
